The activity diagram below shows the activity flow that follows a submission of CTM Data by the Pipeline Operator to the AEMO STTM systems. Upon receipt of the transaction file, the STTM
systems will perform primary validation (file name) and provide a Message Acknowledgment back to the submitter indicating that the file is valid or that it is in error. If the file is found to be invalid during primary validation, no further action is taken by the STTM system. If the file is found to be valid during the primary validation, the STTM system will proceed to validate the data within the file and provide a Transaction Acknowledgement back to the submitter indicating whether the file passed data validation or not.
