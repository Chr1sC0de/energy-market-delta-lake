The OMP-USR will be rejected only immediately after submission and user will know about the reason why the OMP-USR was rejected because the user will receive an transaction acknowledgement with the event code = reason why it was rejected.
The only reason why a request previously submitted by a user doesn't appear in  the  OMP-STATUS (is not matched approved) is that the total  quantity requested from the SSP was not confirmed by the SSP.
When a SSP submits a confirmation, but user doesn't, then the SSP is not going to receive any line in the OMP-STATUS report for the user.
