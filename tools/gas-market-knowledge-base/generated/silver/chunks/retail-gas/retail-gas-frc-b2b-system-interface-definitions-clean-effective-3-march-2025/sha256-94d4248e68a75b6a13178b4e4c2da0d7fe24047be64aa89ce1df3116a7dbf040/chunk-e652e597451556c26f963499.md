A User issues a NMIStandingDataRequest when they have a known MIRN and wish to retrieve the standing data for it.
The diagram below shows the sequence of events for this transaction:
Figure 4-54 NMI Standing Data Request Sequence Diagram
1, AseXML Transaction = NMIStandingDataRequest. 1, From Object = User. 1, To Object = FRC Hub. 1, Process Flow = MIRN 4A. 2, AseXML Transaction = NMIStandingDataRequest. 2, From Object = FRC Hub. 2, To Object = NO/ Network Operator. 2, Process Flow = MIRN 4A
The Network Operator will attempt to obtain the MIRN Standing Data relevant to the supplied MIRN.
In SA, not all MIRN Standing Data may be available regarding additional charges applicable to the MIRN within the required turnaround time for the transaction.  In this case, two MIRN Standing Data Responses will be issued, the initial response containing an indicator to the User that more data is to follow.  The second response follows when the additional data becomes available.
