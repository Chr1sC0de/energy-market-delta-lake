A Network Operator sends a CustomerDetailsRequest to the current Retailer when they reasonably believe that the information in the CustomerDetailsNotification has not been previously provided in a Notification transaction or that the information they hold is or may be incorrect.
A Network Operator must only send a maximum of one CustomerDetailsRequest per MIRN per day The following table shows the aseXML transaction.
ASEXML TRANSACTION, 1 = TABLE OF TRANSACTIONS. ASEXML TRANSACTION, 2 = TABLE OF TRANSACTIONS. Transaction Name, 1 = Ref No. Transaction Name, 2 = Transaction Type. CustomerDetailsRequest, 1 = 72. CustomerDetailsRequest, 2 = Customer Details Request from DB
These business transactions will be mapped to the new 'CUST' Transaction Group in aseXML.
1, ASEXML TRANSACTION = CustomerDetailsRequest. 1, FROM OBJECT = Network Operator. 1, TO OBJECT = FRC Hub. 1, PROCESS FLOW = . 2, ASEXML TRANSACTION = CustomerDetailsRequest. 2, FROM OBJECT = FRC Hub. 2, TO OBJECT = Retailer. 2, PROCESS FLOW =
