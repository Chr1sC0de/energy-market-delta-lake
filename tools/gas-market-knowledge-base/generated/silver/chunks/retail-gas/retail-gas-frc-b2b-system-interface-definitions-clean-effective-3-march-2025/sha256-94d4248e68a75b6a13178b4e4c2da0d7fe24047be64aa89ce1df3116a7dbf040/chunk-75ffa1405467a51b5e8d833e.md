As defined in the B2B-B2M Hub Specifications and Architecture documents all date/time stamps in the messaging layer (ebXML) and in the headers of aseXML transactions will be expressed in GMT+10 (market time).
In the following transaction specifications, where a data element is optional and data is not going to be provided for a specific use of the transaction, then that data element should not be included in the transaction.  Specifically, the inclusion of the data element with no data or with a '0' should not be used as a way of indicating that there is no data to be communicated.
