If:
- (a) a prospective FRO delivers a transfer request in relation to a delivery point (' new request ') to AEMO ;
- (b) a transfer request in relation to that delivery point (' existing request ') has already been delivered to AEMO (whether by that or any other User ); and
- (c) the existing request is being processed by AEMO but the User who delivered the existing request to AEMO has not yet been registered in the AEMO metering database as the FRO for the delivery point to which the existing request relates,
then AEMO must not process the new request and must, by midnight on the next business day after the day on which the new request was delivered to AEMO , deliver a notice stating this fact to the User who delivered the new request.
