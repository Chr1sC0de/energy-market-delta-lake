The server returns a Content-Type of application/json, and a JSON formatted string consisting of two fields: status and error. The content of these fields is described in Table 4.
Table 4 Response fields

Field, 1 = Data Type. Field, 2 = Description. Data, 1 = Object. Data, 2 = This data object contains all the results of the submission. The properties of the data object are dependent on the service call.
An example of a successful report request response is shown below:
```
Content-Type: application/json
```
```
HTTP/1.1 200 OK Content-Length: length {"data":{}}
```
