What the repository demonstrates

These claims are limited to behavior documented in the repository: local and AWS-oriented service boundaries, Dagster data assets, storage resources, and protected route handling.

Source-to-lake ingestion

Source discovery, unzipper flows, bronze ingestion, and archive handling move selected AEMO and NEMWeb inputs toward Delta-backed current-state tables.

Orchestrated data platform

Dagster webservers, daemon, PostgreSQL state, and the aemo-etl code location form the runtime boundary for assets, schedules, sensors, and jobs.

Exploration and access

Caddy exposes the static root page, the guest Dagster route, protected operator routes, and the local Marimo notebook service without adding another frontend runtime.
