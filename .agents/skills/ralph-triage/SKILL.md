# Ralph Triage

Use this skill to prepare GitHub Issues for the repo-local Ralph loop. Triage is
the gate before `$ralph-loop drain`.

## Read First

- Read `CONTEXT.md` for canonical terms, especially **Delivery mode**,
  **Integration target**, **Local integration**, and **Promotion**.
- Read `docs/agents/triage-labels.md` for the label vocabulary.
- Read `docs/agents/ralph-loop.md` before changing Ralph workflow behavior.
- Use `gh` for issue reads and writes.

## Goal

Each triaged issue should have:

- Exactly one category label: `bug` or `enhancement`.
- Exactly one state label: `needs-triage`, `needs-info`, `ready-for-agent`,
  `ready-for-human`, or `wontfix`.
- At most one **Delivery mode** label: `delivery-gitflow` or `delivery-trunk`.

Use `ready-for-agent` only when the issue is complete enough for Ralph to drain
without more human judgment. A ready issue must include:

- `## What to build`
- `## Acceptance criteria`
- `## Blocked by`

## Delivery Mode

- Default to `delivery-gitflow`.
- Use `delivery-trunk` only for clearly small docs, tests, tooling, or script
  changes with low runtime risk.
- Keep `delivery-gitflow` for runtime behavior, infrastructure, Dagster, S3,
  LocalStack, cross-**Subproject** work, broad refactors, or unclear scope.
- If both delivery labels are present, keep `delivery-gitflow` and remove
  `delivery-trunk`.

## Workflow

1. Fetch the issue body, labels, and comments.
2. Resolve conflicting state labels before making other changes. If the right
   state is ambiguous, ask the maintainer.
3. Add the category, state, and **Delivery mode** labels.
4. If the issue is not ready, comment with the missing information or reason.
5. For manual triage, if the issue is ready, make the body match the
   ready-for-agent contract when the needed details are already present in the
   issue or comments.

Every triage comment must begin with:

```markdown
> *This was generated by AI during triage.*
```

Automated Ralph triage may label, comment, or close issues, but must not edit
repo files or rewrite issue bodies unless the automated prompt explicitly allows
it. Ralph v1 does not let automated triage write `.out-of-scope/` records. If
an enhancement looks like `wontfix` and needs an out-of-scope record, mark it
`ready-for-human` instead and explain why.
