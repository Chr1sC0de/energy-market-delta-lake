# Ralph Issue Refresh

Use this skill for **Ready issue refresh** in the Ralph drain. Refresh compares
open GitHub Issues with the current **Integration target** after successful
**Local integration** and before Ralph claims the next `ready-for-agent` issue.

## Read First

- Read `CONTEXT.md` for canonical repo language.
- Read `docs/agents/ralph-loop.md#ready-issue-refresh` for the contract.
- Mutate only GitHub Issue metadata. Do not change files, commits, branches,
  **Local integration**, or **Promotion** state as part of refresh.

## Allowed Updates

Refresh may update:

- issue comments
- issue body context
- triage, state, Ralph runtime, or **Delivery mode** labels
- completed closure for issues already satisfied or obsolete on the current
  **Integration target**

Every refresh comment must begin with this exact audit prefix:

```markdown
> *This was generated by AI during Ready issue refresh.*
```

## Workflow

1. Identify issues that are open and relevant to the just-integrated change.
2. Compare each issue body, blockers, labels, and acceptance criteria to the
   current **Integration target**.
3. Keep still-actionable issues `ready-for-agent` only when they still contain
   `## What to build`, `## Acceptance criteria`, and `## Blocked by`.
4. Move unclear stale issues to `needs-triage` with evidence.
5. Close only issues clearly satisfied or obsolete on the current branch, and
   include evidence in the refresh comment.

Do not close unclear work as completed. If both `delivery-gitflow` and
`delivery-trunk` are present, keep `delivery-gitflow` unless the issue
explicitly requires another **Delivery mode**.
